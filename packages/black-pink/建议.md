# black-pink 后续强化建议

## 当前短板

按现在这版，问题不在于能不能动，而在于短视频里的记忆点和完成度还不够。

- 交互单一。现在基本是鼠标扫到一列，这一列上下飞走，观众很快就能看懂，后续没有新的视觉信息。
- 画面太平。当前逻辑主要只有位移，缺少缩放、透明度、旋转、分层延迟，整体厚度不够。
- 节奏偏平均。统一的 `1s ease-out` 不够利落，画面容易显得拖。
- 主体不明确。现在更像背景切片在运动，没有一个明确的文字、图形或 logo 作为焦点。
- 收尾不完整。残影和状态复位问题会让成片显得像 demo，而不是完成品。

## 强化方向

### 1. 交互从单列升级为中心扩散

不要只处理当前列，改成以鼠标点为中心，按距离计算影响范围。

- 靠近鼠标的方块位移更大
- 远处方块只做轻微扰动
- 动效看起来更像冲击波，而不是表格位移

这是最值得优先做的一步，改完后质感会明显提升。

### 2. 增加多维变化

每个方块不要只做 `translateY`，至少叠加三层变化：

- `translate`
- `rotate`
- `opacity`

原因：

- 位移负责结构变化
- 旋转负责碎裂感
- 透明度负责消散感

只有位移时，效果容易像拼图错位，不像精心设计的视觉动画。

### 3. 加时序，不要同时动

可以按照方块与触发点的距离做 stagger。

- 近处先动
- 远处后动
- 延迟可以控制在 `30ms` 到 `200ms`

这样会出现层层传导的感觉，更适合短视频传播。

### 4. 做完整闭环

现在主要是在设置新状态，但没有彻底清理旧状态。后续要把动效做成完整闭环。

- 鼠标进入时触发爆裂
- 鼠标移动时更新影响范围
- 鼠标离开时统一复位
- 可以考虑让部分方块回弹，而不是所有方块都直接消失

如果没有闭环，最后画面容易留下残影，看起来不够干净。

### 5. 做一个明确主体

当前只是背景切片在动，缺少内容中心。建议加入一个能被观众记住的主体。

可选方向：

- 拼出 `BLACKPINK` 文字
- 拼出心形或蝴蝶等图形
- 拼出一张高对比头像或剪影
- 拼出 logo 或专辑感图形

有主体之后，动效就不再只是技术展示，而是一个有内容的短视频镜头。

### 6. 增加第二层视觉元素

可以在原图层之上再叠一层辅助效果，例如：

- 描边层
- 高光层
- 残像层
- 轻微噪点
- RGB 偏移

这样飞散时不只是像素块在动，而是会有更强的氛围感。

### 7. 拆开过渡曲线

不要所有属性都共用一条过渡曲线。

建议：

- 位移用更有冲劲的曲线
- 透明度用更快的衰减
- 旋转单独设定节奏

统一 `ease-out` 容易显得廉价，拆开之后层次会更丰富。

### 8. 增加未触发态的呼吸感

静止状态不要完全死掉，可以加入极轻微的动态：

- 微弱漂浮
- 轻微缩放
- 扫描线
- 噪点闪烁

这样第一帧就不会太空，录屏时也更有氛围。

## 推荐升级路线

### 路线 1：冲击波碎片风

特点：

- 保留当前切片结构
- 从按列触发改成按半径触发
- 增加旋转、透明度和分层延迟

优点：

- 改动最小
- 观感提升最大
- 最适合基于当前代码继续迭代

### 路线 2：文字显影风

思路：

- 初始状态是碎片悬浮
- 鼠标划过后逐步拼出文字或图形
- 最后再炸开或淡出

优点：

- 前后反差更明确
- 更容易形成传播点

### 路线 3：玻璃故障风

思路：

- 在位移基础上加入 RGB 偏移
- 加错位扫描
- 加轻微模糊和闪烁

优点：

- 风格更强
- 更接近 MV / 封面视觉

缺点：

- 实现复杂度更高

## 推荐实现优先级

建议按这个顺序推进：

1. 修复残影和复位逻辑，先把交互闭环做完整。
2. 从单列触发改成中心扩散。
3. 增加 `opacity`、`rotate` 和 stagger。
4. 加入明确主体，例如文字或图形。
5. 最后再补噪点、高光、故障感等氛围层。

## 面向抖音的成片建议

如果目标是发抖音，建议按短视频镜头逻辑来设计，而不是只从代码层面思考。

- 时长控制在 `6` 到 `10` 秒
- 第一秒内要出现爆点
- 交互不要反复乱划，保留一次最强触发即可
- 结尾要停住，让最终画面能被看清

推荐节奏：

1. 静止 `0.5s`
2. 触发爆裂
3. 聚合成文字或图形
4. 停住 `1s`

## 下一步建议

如果继续做，不建议再只修局部小 bug。更有效的方向是直接升级为这一套：

- 中心扩散
- stagger 延迟
- 透明度变化
- 旋转变化
- 回弹或复位闭环

这几项做完，整体观感会比现在高一个层级。
